# ⚡ FTC ATEM Auto-Switcher

Automatically switch Blackmagic ATEM camera inputs between FTC competition fields based on live match progress. No more manual camera switching during events — just set it up and let it run.

**Works with all Blackmagic ATEM switchers** — Mini, Mini Pro, Mini Extreme, Television Studio, Constellation, and more.

---

## Quick Start

### 1. Download

Download `FTC-ATEM-Switcher-Setup.exe` from the [Releases](https://github.com/soriiku/NYCFIRST-FTC-ATEM-Switcher/releases) page.

### 2. Get FTC API Credentials

You need an FTC Events API account to pull match data:

1. Go to **[ftc-events.firstinspires.org/services/API](https://ftc-events.firstinspires.org/services/API)**
2. Register for an account
3. Save your **username** and **authorization key**

### 3. Run the App

Double-click `FTC-ATEM-Switcher.exe`. Everything is bundled inside — no installation needed.

On your first launch, a sleek setup screen will appear directly in the app to ask for your Event Code, API keys, ATEM IP, and camera inputs. Once filled out, the main switcher dashboard loads automatically.

> **Tip:** If you ever need to completely reset the configuration to start fresh, just delete the invisible `.env` file that appears next to the `.exe` and restart the app.

## Dashboard Features

The web dashboard at `localhost:3000` gives you full control:

| Section | What it does |
|---|---|
| **Active Camera** | Shows which field camera is active, with manual override buttons |
| **Match Progress** | Progress bar showing completed vs. total matches |
| **ATEM Status** | Connection status, IP input to connect to any ATEM switcher |
| **HDMI Input Mapping** | Set which HDMI input on the ATEM corresponds to each field |
| **Switch Mode** | Choose how the camera switches (see below) |
| **Event Configuration** | Change the FTC event, season, API credentials at any time |
| **Match Schedule** | Full schedule with scores, filterable by status |
| **Event Log** | Real-time log of all events and switches |

**All settings can be changed live from the dashboard — no restart needed.**

## Switch Modes

You can choose between two camera switching methods from the dashboard:

### Score-Based (Default)

The camera switches to the next field **after a match's score is committed** by the scorekeeper. This is the most reliable method — the switch happens exactly when the match result is finalized.

### Timer-Based (2:45)

The camera switches exactly **2 minutes and 45 seconds after a match starts**. This ensures the switch happens during the match, giving the audience a smooth transition before the next match begins on the other field. Best for events where scores tend to be committed late.

---

## Automatic Playoff Detection

When all qualification matches are complete, the app automatically:

1. Checks the FTC API for a playoff schedule
2. If found, switches to playoff mode and loads the new matches
3. If not posted yet, retries every 30 seconds until it appears
4. Continues tracking through all playoff rounds (semis, finals, etc.)

New playoff rounds are picked up automatically as they're added — no manual intervention needed for the entire event.

## Network Setup

Make sure your computer running the switcher is on the **same network** as the ATEM:

```
Computer (running this app)  ←→  Network Switch  ←→  ATEM Switcher
                                       ↕
                                   Internet
                              (for FTC API access)
```

Your computer needs:
- **Internet access** to reach the FTC Events API
- **Local network access** to reach the ATEM switcher's IP

## For Developers

If you'd prefer to run from source instead of the exe:

### Prerequisites

- [Node.js](https://nodejs.org/) (LTS version recommended)

### Install & Run

```bash
git clone https://github.com/soriiku/NYCFIRST-FTC-ATEM-Switcher.git
cd NYCFIRST-FTC-ATEM-Switcher
npm install
npm run electron:dev
```

### Dry Run Mode (No ATEM)

Test the full dashboard without connecting to an ATEM:

```bash
npm run dry-run
```

### Build the Executable

```bash
npm run electron:build
```

Output: `dist/FTC-ATEM-Switcher.exe`

## Configuration Reference

These are the `.env` values (auto-generated by the setup wizard):

| Variable | Description | Default |
|---|---|---|
| `FTC_USERNAME` | FTC Events API username | *(required)* |
| `FTC_AUTH_KEY` | FTC Events API auth key | *(required)* |
| `FTC_EVENT_CODE` | Event code from FTC | *(required)* |
| `FTC_SEASON` | Competition season year | `2025` |
| `FTC_TOURNAMENT_LEVEL` | `qual` or `playoff` | `qual` |
| `ATEM_IP` | Your ATEM's IP address | `192.168.10.240` |
| `FIELD1_INPUT` | ATEM input number for Field 1 | `1` |
| `FIELD2_INPUT` | ATEM input number for Field 2 | `3` |
| `POLL_INTERVAL_MS` | How often to check for updates (ms) | `5000` |
| `DASHBOARD_PORT` | Dashboard web server port | `3000` |

> All `.env` values are starting defaults — you can override everything from the dashboard at runtime.

## Troubleshooting

| Problem | Solution |
|---|---|
| "FTC API connection failed" | Check your username/auth key. Make sure you have internet access. |
| "ATEM connection timed out" | Verify the ATEM IP and that you're on the same network. |
| Camera not switching | Check the Event Log on the dashboard for errors. Make sure matches are being scored. |
| Wrong camera on each field | Update the HDMI Input Mapping in the dashboard settings. |
| Want to change event | Use the Event Configuration panel on the dashboard, or delete `.env` and restart. |
| Dashboard won't load | Make sure port 3000 isn't in use. Check if another instance is already running. |
